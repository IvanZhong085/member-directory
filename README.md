# 會員名錄網站

純靜態的一頁式會員名錄（白底紅襯色）。公開名錄是**唯讀**的；只有知道**管理密碼**的人能進入編輯模式、發布更新。GitHub 權杖本人放在一個小型雲端中介服務（Cloudflare Worker）裡，**從頭到尾不會出現在任何瀏覽器裡**——密碼登入後由伺服器代為發布，任何人（含小朋友）在任何電腦，只要密碼對就能編輯。

## 檔案結構

```
member-site/            ← 這個資料夾的內容 = 網站根目錄
├─ index.html    公開名錄（總覽 → 分組 → 成員內頁 + 搜尋）
├─ styles.css
├─ app.js
├─ admin.html    編輯頁（密碼登入、改資料、換照片、增刪成員、一鍵發布）
├─ admin.js
├─ data.js       ★ 全部資料都在這一個檔（分組與成員，人數隨增刪變動）
├─ images/       成員照片
├─ spotlight.html / spotlight.js   夥伴聚光燈產生器（貼文文案＋分享卡）
├─ sharecard.js  分享工具模組（分享卡繪製、QR、vCard、文案模板）
├─ qrcode.js     QR code 產生函式庫（qrcode-generator，MIT 授權，離線可用）
```

## 一、部署到 GitHub Pages（設定一次）

1. 在 GitHub 建立一個**新的公開 repo**（例如 `member-directory`）。
2. 把 **`member-site/` 裡面的所有檔案**放到這個 repo 的**根目錄**（讓 `index.html` 在最上層），推上去。
3. repo → **Settings → Pages** → Source 選 **Deploy from a branch** → Branch 選 `main`、資料夾 `/(root)` → Save。
4. 等一下，網站會出現在 `https://<你的帳號>.github.io/<repo 名稱>/`。把這個網址存成書籤。

> 建議用「新的、獨立的公開 repo」，不要和其他專案混在一起。

---

## 二、開啟一鍵發布（設定一次，由大人做）

這一步做完後，之後任何人只要輸入**管理密碼**就能編輯並發布，完全不用碰 GitHub 權杖、檔案或 Git 指令。這一步分兩段：先建 GitHub 權杖，再照 **`worker/README.md`** 的教學把它交給一個免費的 Cloudflare Worker 保管。

1. 到 GitHub → 右上頭像 → **Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token**。
2. 設定：
   - **Repository access** 選「Only select repositories」→ 只勾**這個名錄的 repo**。
   - **Permissions → Repository permissions → Contents** 設為 **Read and write**。
   - 其他不用動，建立後**複製權杖**（只會顯示一次）。
3. 打開 **`worker/README.md`**，跟著步驟在 Cloudflare（免費帳號即可）建立一個 Worker，把上面的權杖、一組你要設定的**管理密碼**貼進去（過程都在網頁上點一點，不需要安裝任何軟體）。
4. 拿到 Worker 網址後，打開名錄網站 → 點右下角**小齒輪** → 「連線設定」→ 貼上網址 → 儲存。

完成後，**GitHub 權杖只存在 Cloudflare 那一端**，任何瀏覽器都不會保管它；操作者只需要知道管理密碼。

> ⚠️ 管理密碼等於編輯權限，不要隨便公開；密碼錯太多次（15 分鐘內 5 次）會自動暫時鎖住，防止亂猜。要換密碼、換權杖，都在 Cloudflare 的 Worker 設定裡改，不用改網站程式碼。

## 三、日常怎麼更新（小朋友也會）

1. 打開名錄網站，點右下角**小齒輪** → 輸入**管理密碼**進入編輯模式。
2. 修改：
   - **加人**：上面輸入框打姓名，按 Enter 就新增一位（可連續加）。
   - **刪人／刪組**：成員卡的刪除鈕、左邊分組滑過的 ✕；刪錯了右下角按「**復原**」就回來。
   - **換照片**：成員卡「更換照片」選圖，會自動縮圖，不用另外處理圖片。
   - **複製**：資料相似的人可用「複製」鈕。
3. 改好後，按右上角紅色「**發布到網站**」。
4. 等大約 1 分鐘，公開網站就更新好了 ✔

> 「下載備份」按鈕會存一份 `data.js` 到電腦，當備份用（平常用不到）。密碼登入後**每 30 分鐘**需重新輸入一次密碼（固定期限，非閒置計時）；若在編輯到一半時到期，畫面會提示重新登入，**草稿資料不會遺失**，登入後再按一次「發布」即可。

## 四、分享夥伴與聚光燈產生器

### 每位成員都是一張數位名片

成員內頁下方有「**分享這位夥伴**」工具列：

- **複製連結**：複製這位成員的專屬網址（`…/m/成員id.html`）。貼到 LINE／FB 會顯示**本人照片與介紹**的預覽卡。
- **QR code**：現場給人掃，直接開啟該成員介紹頁（跨會拜訪、擺攤、名片背面都好用）。
- **存入通訊錄**：下載 vCard（.vcf），一鍵把姓名、行業、服務摘要與介紹連結存進手機聯絡人。
- **貼文／分享卡**：跳到聚光燈產生器並自動帶入這位成員。
- 手機瀏覽器另有系統原生「分享…」按鈕。

### 聚光燈產生器（spotlight.html）

網站管理員的每週發文工作台：選一位夥伴，立刻得到——

1. **三種貼文文案**：FB／IG 版、正式版、LINE 短版（模板直接可用；發文前請過目）。
2. **AI 潤飾提示詞**：一鍵複製整包夥伴資料＋寫作要求，貼給 ChatGPT／Claude 產出更活的版本，AI 不會拿到資料以外的內容。
3. **分享卡圖片**：白底紅襯設計、含照片／標語／服務／引薦對象／QR code，直式 4:5 與方形 1:1 兩種尺寸，PNG 下載即可發文。

每週輪播就按「下一位 →」，順序照名錄分組排列。

### 分享預覽頁（m/ 資料夾）

`m/成員id.html` 是給 LINE／FB 爬蟲看的小頁面（og 標籤），真人開啟會自動跳轉回名錄內頁。**名錄有增刪成員後**，在 repo 根目錄執行：

```
node tools/build-member-pages.mjs
```

會重新產生全部成員頁並清除失效頁，跟著一起發布即可。只改姓名以外的資料（服務、照片等）不重跑也能用，但預覽文字會是舊的，建議順手重跑。

## 五、名冊管理：CSV 匯出入與批次照片

後台工具列有三顆按鈕，給「常常要核對人數、調整分組」的網管用：

### 匯出 CSV

一鍵下載全會名冊（Excel 直接開，中文正常）。欄位：編號、姓名、行業職稱、分組代號、分組名稱、服務項目、適合引薦對象、宣傳標語、所屬公司、主要營業項目、照片、資料需確認、刪除。多項欄位（服務等）以 `|` 分隔。

### 匯入 CSV（批次更新組別與欄位）

用匯出的檔案當範本，改好再上傳。**套用前一定會先顯示差異預覽**（誰移組、誰新增、誰刪除、各組人數前後對照），按「套用變更」才會生效，套用後仍可用「上一步」復原，最後記得按「發布到網站」。

規則（防呆設計）：

- **比對鍵**：先用「編號」找人，找不到才用「姓名」；同名又無編號可判定的列會列為錯誤、不套用。
- **空格＝不變更**；想清空某欄，填一個 `-`。
- **移組**：把「分組代號」改成目標組即可；代號不存在但同列有填「分組名稱」＝建立新組。
- **刪人一定要明確標記**：在「刪除」欄填「是」。漏列、整列刪掉都**不會**刪人。
- 「照片」欄僅供核對，匯入不會動照片。

### 批次照片

一次選多張圖，檔名自動配對成員（擇一即可）：**編號**（`001.jpg`、`079小明.jpg` 開頭是編號也行）、**姓名**（`曾俊凱.jpg`）、或成員 id。配對結果先預覽再套用，照片會自動置中裁成名錄比例。

> ℹ️ **照片儲存**：每次發布後，GitHub Action 會在一兩分鐘內自動把內嵌照片轉成 `images/` 實體圖檔、重建 `m/` 分享預覽頁（見下一節）。Worker 升級（見 `worker/README.md` 的「升級」一節）可以讓這件事在發布當下就完成，屬於選配加速。

## 六、Google 試算表每日同步（roster.csv）

網站每次發布後，GitHub Action 會自動做三件事：**內嵌照片轉實體檔、重建分享預覽頁、更新名冊鏡像 `roster.csv`**。名冊鏡像的固定網址：

```
https://ivanzhong085.github.io/member-directory/roster.csv
```

### 接到 Google 試算表（設定一次，約一分鐘）

1. 開一個新的 Google 試算表。
2. 在 `A1` 貼上：`=IMPORTDATA("https://ivanzhong085.github.io/member-directory/roster.csv")`
3. 完成。之後名錄一有發布，試算表會**自動跟上**（Google 約每小時重抓一次，至少每日更新）。

用途：核對人數與分組、製作**分會產業小組 PDF**（可另開分頁用 `FILTER`／樞紐整理版面後「檔案 → 下載 → PDF」）、給幹部唯讀名冊。

### 注意

- 試算表是**唯讀鏡像**——在上面改字不會改到網站。要改資料：複製內容到新分頁改好 → 下載 CSV → 後台「匯入 CSV」→ 發布（欄位格式與匯入完全相容）。
- `roster.csv` 不含 BOM（Google 取向）；要用 **Excel** 開名冊請改用後台的「匯出 CSV」（含 BOM，中文不亂碼）。
- 名錄增刪成員後**不再需要**手動跑 `node tools/build-member-pages.mjs`，Action 會自動處理；本機開發時仍可手動執行。

## 七、夥伴自填表單（Google 表單）

讓夥伴自己補資料、傳照片，網管審查後進名錄。**必填只有四項：姓名、所屬公司、主要營業項目、同意聲明**；其餘（編號、專業別、服務項目、引薦對象、標語、公司網站、形象照、名片、商品照 ≤5）皆選填，**留白＝沿用名錄現有資料**（與「匯入 CSV」規則一致）。

1. **建表**：把 `tools/google-form.gs` 貼到 script.google.com 新專案，執行 `createMemberForm`；再照腳本開頭註解**手動補 3 題檔案上傳**（Google 不開放程式建立上傳題，約 2 分鐘）。
2. **接上催收**：把表單網址填入 `admin.js` 最上方的 `FORM_URL`，後台「缺資料」按鈕產生的催收訊息就會自動附連結。
3. **收到回應後**：執行 `refreshImportTab` → 下載「匯入用」分頁 CSV → 後台「匯入 CSV」→ 發布；執行 `renameUploads` → 照片自動改名「編號_姓名_形象照.jpg」→ 下載後丟「批次照片」（縮圖逐張審查）；名片與商品照用各成員卡的上傳鈕放入。

## 資料欄位（data.js）

每個分組：`code`（代號）、`name`（名稱）、`leader`（組長）、`members`（成員陣列）。

每位成員：
| 欄位 | 說明 |
|------|------|
| `number` | 編號 |
| `name` | 姓名 |
| `title` | 行業／職稱 |
| `services` | 服務項目（一項一行） |
| `targets` | 適合引薦對象 |
| `tagline` | 宣傳標語 |
| `image` | 照片：檔名或內嵌圖片；空＝無照片 |
| `company` | 所屬公司（預留，待補充） |
| `business_items` | 主要營業項目（預留，待補充） |
| `dataIssue` | `true` 時前台顯示「資料需確認」 |
