發布設定（設定一次就好）

填好之後，之後只要按「發布到網站」就會自動更新公開網站，不用碰檔案或 Git。

權杖請到 GitHub → Settings → Developer settings → Fine-grained tokens 建立，只授權「這一個 repo」的 Contents：Read and write。
權杖會用你設定的管理密碼加密後才存在這台電腦的瀏覽器——之後日常編輯只要輸入密碼，不必再碰權杖；沒有密碼的人打不開、也發布不了。權杖若外流，到 GitHub 撤銷（Revoke）即可作廢。

專業分組
資料安全說明：這個頁面的編輯只會暫存在你自己的瀏覽器裡，不會影響公開網站；公開資料只有持「管理權杖」的管理員按下「發布到網站」才會更新，其他人無法篡改。

如何更新網站？ 在這裡改好後，按右上角紅色「發布到網站」即可（首次需在「設定」貼上管理權杖）。「下載備份」可另存一份 data.js 備用。
  1. 快速新增成員：在成員列表上方輸入姓名，按 Enter 就新增一位（可連續輸入連續加）；也可用「+ 新增成員」加空白的再慢慢填。
  2. 刪除成員／分組：成員卡右上的刪除鈕、左側分組滑過出現的 ✕。刪除後右下角會跳出「復原」按鈕，按錯可立即救回，不會跳確認框卡你。
  3. 複製成員：卡片右上的複製鈕，會複製一份接在後面（適合資料相似的人）。
  4. 換照片：在成員卡片點「更換照片」選圖檔即可（會自動縮圖，直接內嵌，不必另外處理圖片檔）。
  5. 新增分組：左側「+ 新增組」或分組列表最下方的虛線「＋ 新增分組」。
  6. 草稿會自動存在瀏覽器，關掉再開仍在；改完務必「下載 data.js」覆蓋原檔才會真正更新到網站。
