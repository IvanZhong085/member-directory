連線設定

這裡填的是「發布服務的網址」，不是密碼、也不是權杖——這台電腦的瀏覽器完全不會保管任何機密。真正的 GitHub 權杖只放在你自己架設的 Cloudflare Worker 裡，瀏覽器只會送密碼過去驗證。

還沒建立 Worker？見 member-site/worker/README.md 的教學（Cloudflare 網站上點一點，不需要安裝任何軟體）。網址設定好、儲存後，回到登入畫面輸入管理密碼即可。

還有尚未發布的變更

你的修改已暫存在這台裝置，不會遺失；但公開的會員名錄現在顯示的還是上一次發布的版本。要先發布再回名錄嗎？

變更預覽

資料安全說明：這個頁面的編輯只會暫存在你自己的瀏覽器裡，不會影響公開網站。真正能改到公開網站的，只有輸入正確管理密碼的人；密碼登入後由伺服器（Cloudflare Worker）代為發布，瀏覽器本身永遠不會拿到、也不會存放 GitHub 的權杖。密碼錯太多次會被暫時鎖住，防止有人亂猜。

如何更新網站？ 輸入管理密碼進入編輯模式，改好後按右上角紅色「發布到網站」即可。「下載備份」可另存一份 data.js 備用。
  1. 快速新增成員：在成員列表上方輸入姓名，按 Enter 就新增一位（可連續輸入連續加）；也可用「+ 新增成員」加空白的再慢慢填。
  2. 刪除成員：成員卡右上的刪除鈕，刪除後右下角會跳出「復原」按鈕，按錯可立即救回。（分組整組刪除已移除，避免誤刪。）
  3. 上一步／下一步：左上角兩個箭頭（或 Ctrl+Z／Ctrl+Y），最多可往前回復 10 步。
  4. 換照片＋裁剪：點「更換照片」選圖後，會跳出裁剪框，拖曳、縮放把人物放進框內再「套用」。
  5. 暫存：按「暫存」把目前進度立即存進這台裝置的瀏覽器（不會發布出去），下次打開還在。
  6. 分組列：在左側；電腦可用 « 收合、手機點左上 ☰ 叫出。新增分組用「+ 新增組」或列表下方虛線鈕。
  7. 改完務必按「發布到網站」才會真正更新到公開網站，光改草稿不會。
